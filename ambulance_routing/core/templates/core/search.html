Algorithm Complexity Reference
Algorithm Preprocessing Search (Best) Search (Worst) Space How it works
KMP O(m) O(n) O(n+m) O(m) Builds failure function from pattern. Never backtracks in text. Linear guaranteed.
Rabin-Karp O(m) O(n+m) O(nm) O(1) Rolling hash of text window. Hash match → verify. Great for multiple patterns.
Naïve/Brute O(1) O(n) O(nm) O(1) Check every position. Backtracks. Worst case with repeated chars.
n = length of text being searched  ·  m = length of search pattern
